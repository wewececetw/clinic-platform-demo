# 06. AI 能力藍圖

## 6.1 Phase 1：智慧症狀分流 ✅

**輸入**：`TriageRequest(ClinicId, Symptoms)`
**輸出**：`TriageResponse(Department, DepartmentId, Priority, EstimatedWaitMinutes, Reasoning)`

### 流程
1. `TriagePromptBuilder` 組裝提示（含診所科別清單、歷史候診資料）
2. `ILlmClient` 呼叫 LLM（Omlx 本地優先，fallback 到 Groq）
3. 解析 JSON 回應 → 對應 `Department`
4. 護理師可採納 AI 建議，或手動覆寫

**抽象層設計**：`ILlmClient` 統一 OpenAI-compatible 介面，支援多 provider。

詳見 [specs/001-ai-triage](../../specs/001-ai-triage/)。

## 6.2 Phase 2：自然語言指令 ✅

### 支援指令（`CommandAction` enum）

| Action | 允許角色 | 範例 |
|--------|---------|------|
| `CallNext` | Nurse, Doctor | 「叫下一位」 |
| `Skip` | Nurse, Doctor | 「3 號過號」 |
| `QueryQueue` | Nurse, Doctor, Pharmacist | 「目前有幾個人在等」 |
| `CompleteConsult` | Doctor | 「結束這位看診」 |
| `CreatePrescription` | Doctor | 「開普拿疼 500mg TID 三天份」 |
| `QueryStats` | Nurse, Admin | 「今天看了幾個病人」 |

### 架構

```
User 自然語言 → CommandPromptBuilder → LLM → { Action, Params }
                                              ↓
                                        CommandRouter
                                        ├─ 權限驗證（AllowedRoles）
                                        └─ Dispatch to Executor
                                              ↓
                                        呼叫對應 Service
```

**雙層權限**：Controller 層驗角色 + Executor `AllowedRoles` 再驗一次（深度防禦）。

詳見 [specs/004-nl-command](../../specs/004-nl-command/)。
