## 使用方式

- **新進開發者**：依序閱讀 01 → 02 → 03 → 04 建立全貌
- **接手某模組**：讀 03 找模組位置 → 05 讀該模組 BR 規則 → `specs/` 讀技術細節
- **規劃新功能**：先讀 06 看 AI 藍圖定位 → 依 SDD 流程（speckit）補 specs

## 與 specs/ 的關係

- `docs/business-logic/`：**WHAT & WHY** — 整體商業邏輯、規則、狀態機
- `specs/NNN-feature/`：**HOW** — 單一功能的 spec.md / plan.md / tasks.md

## 維護原則

- 新增功能或修改業務規則時請同步更新此處
- 業務規則採 `BR-<模組>-<編號>` 命名，新增時延續編號不可複用
- Enum 值變更時需同步更新 §04 狀態機圖
